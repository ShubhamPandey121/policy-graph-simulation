# Policy Impact Simulator

A comprehensive, full-stack, AI-driven platform designed to analyze, predict, and visualize the complex ripple effects of policy decisions across various economic, social, and technological sectors.

## 🚀 Overview

The Policy Impact Simulator empowers policymakers, analysts, and researchers to understand the multi-dimensional consequences of a given policy. By combining Large Language Models (LLMs), causal inference, and graph database technologies, the system moves beyond basic text summarization to provide a dynamic, interconnected map of policy implications.

## ✨ Key Features

- **Multi-Agent Simulation Committee**: Policies are evaluated by a specialized team of AI agents, each with a distinct persona (Global Analyst, Economic Strategist, Social Advocate, Causal Scientist, and Real-Time Analyst). This ensures a holistic review of any proposal.
- **Knowledge Graph Visualization**: Powered by Neo4j and Cytoscape.js, the system dynamically maps direct impacts and secondary "ripple effects," allowing users to visually trace how a change in one sector affects others.
- **Advanced Document Scrutinization**: Supports direct uploads of policy PDFs and images. Utilizing LangChain, FAISS, and Tesseract OCR, the system can parse, chunk, and retrieve relevant context from lengthy documents using Retrieval-Augmented Generation (RAG).
- **Causal Inference Engine**: Integrates `EconML` to provide quantitative causal impact analysis (e.g., the statistical effect of funding changes on sector growth), bridging the gap between qualitative LLM analysis and hard statistics.
- **Context-Aware Assistant**: A persistent, floating chat interface that retains the context of the current simulation, allowing users to interrogate the generated graphs and reports interactively.
- **Persistent History**: All simulations and chat histories are securely stored in a MongoDB database, ensuring no analysis is lost across sessions.

## 🏗️ Architecture

### Backend (Python)
- **Framework**: FastAPI + Uvicorn
- **AI Integration**: Groq/OpenAI APIs (Default: `llama-3.1-8b-instant` for large context windows)
- **Vector Search & RAG**: SentenceTransformers (`all-MiniLM-L6-v2`, `CrossEncoder`), FAISS
- **Data Stores**: Neo4j (Graph), MongoDB (Conversations & Sessions)
- **Data Science**: Pandas, EconML, Scikit-learn
- **Document Processing**: PyPDFLoader, pytesseract, pdf2image

### Frontend (React)
- **Framework**: React 19 + Vite
- **Graphing & UI**: Cytoscape.js, Lucide React, Marked.js
- **Styling**: Custom CSS with responsive design, glassmorphism, and dynamic dark/light mode theming.
- **State Management**: React Hooks + Axios for API communication.

## ⚙️ Prerequisites

Before you begin, ensure you have met the following requirements:
- Node.js (v18+)
- Python (3.10+)
- Access to a Neo4j instance (AuraDB or local)
- Access to a MongoDB instance (Atlas or local)
- An API Key from Groq or OpenAI
- *Optional*: Tesseract OCR installed on your system (for image/scanned PDF processing)

## 🛠️ Installation & Setup

### 1. Clone the repository
```bash
git clone <repository-url>
cd policy-graph-simulator
```

### 2. Backend Setup
Navigate to the backend directory and set up the Python environment:
```bash
cd backend
python -m venv venv

# On Windows
venv\Scripts\activate
# On macOS/Linux
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

Create a `.env` file in the `backend` directory (refer to `.env.example` if available):
```env
OPENAI_API_KEY=your_api_key_here
OPENAI_BASE_URL=https://api.groq.com/openai/v1
MODEL_NAME=llama-3.1-8b-instant

# Neo4j and MongoDB URIs should also be configured here
```

Start the FastAPI server:
```bash
uvicorn server:app --reload
```
*The API will be available at `http://localhost:8000`.*

### 3. Frontend Setup
Open a new terminal window, navigate to the frontend directory, and install the Node dependencies:
```bash
cd frontend
npm install
```

Start the Vite development server:
```bash
npm run dev
```
*The application will be available at `http://localhost:5173`.*

## 📖 Usage

1. **Initiate an Analysis**: On the welcome screen, type a query (e.g., "Impact of a 10% tax increase on renewable energy") OR upload a policy document (PDF/PNG).
2. **Review the Simulation**: Wait for the multi-agent system to process the input. Once complete, you will be presented with an interactive graph showing the affected sectors and edges representing the relationships.
3. **Read Agent Reports**: Scroll through the detailed reports generated by the different AI personas to understand the qualitative implications.
4. **Interrogate the System**: Open the chat widget in the bottom right to ask specific follow-up questions about the current simulation.
5. **Session Management**: Use the sidebar to navigate between past simulations or start a new one. All progress is saved automatically.
